# SK텔레콤 AICC (AI Contact Center) 상담사 데모 — 아이디어 요청 프롬프트

아래 프롬프트를 다른 LLM(GPT, Gemini 등)에 그대로 붙여넣어 사용하세요.

## 프롬프트 시작

```
당신은 통신사 고객 경험(CX) 혁신 전문가입니다.

아래는 SK텔레콤이 개발 중인 AI Contact Center(AICC) 상담사 대시보드 데모의 현재 구현 상태입니다. 이 내용을 정확히 이해한 뒤, 맨 아래의 요청사항에 답해주세요.

## 배경

SK텔레콤은 2025년 USIM 해킹 사태를 겪었습니다. 대규모 고객 정보 유출 우려로 고객 신뢰가 크게 훼손되었고, 현재 신뢰 회복이 최우선 과제입니다. 이 AICC 데모는 AI 기술을 활용해 상담 품질을 높이고, 궁극적으로 고객 신뢰를 회복하는 것이 목표입니다.

## 현재 서비스 구조

### 기술 스택
- Frontend: Next.js 16 (App Router, Turbopack), React 19, TypeScript, Tailwind CSS
- UI: Shadcn/UI + Radix UI 컴포넌트
- 상태관리: Zustand (5개 스토어: 인증, 통화, 대화, AI분석, UI)
- AI/음성: OpenAI (GPT-4o-mini, Whisper, Realtime API, TTS)
- DB: Supabase (PostgreSQL, 세션/대화 저장)
- 지식베이스: 44개 마크다운 파일 (요금제, 로밍, 멤버십, 디바이스, 매장 등)

### 페이지 구성
1. `/login` — 상담사 로그인 (데모용 mock 인증)
2. `/dashboard` — 메인 상담 화면 (3컬럼 레이아웃)
3. `/advanced` — 고객 서비스 관리 (요금제·부가서비스·할인·로밍 + 해지 분석 + 요금 시뮬레이터)
4. `/history` — 과거 상담 이력 목록
5. `/history/[id]` — 개별 상담 상세 보기

## 현재 구현된 기능 상세

### 1. 실시간 음성 인식 (STT) — 3가지 모드

**Mock 모드 (데모)**
- 사전 녹음된 시나리오 자동 재생 (요금제 변경 상담)
- 고객/상담사 14개 대화 교환
- 데모 발표용

**Realtime 모드 (OpenAI Realtime API)**
- WebRTC 기반 실시간 오디오 스트리밍
- OpenAI Whisper-1 모델로 한국어 실시간 전사
- 고객/상담사 각각 독립 마이크 입력
- 중간 결과(interim) + 최종 결과(final) 실시간 표시
- 에코 캔슬레이션, 노이즈 서프레션 적용

**AI 대화 모드 (Web Speech API + GPT)**
- Chrome Web Speech API로 음성 인식
- 고객 발화 → GPT-4o-mini가 상담사 응답 자동 생성
- 연속 대화 루프 (듣기 → AI응답 → 다시 듣기)
- Supabase에 세션/대화 자동 저장

### 2. 실시간 대화 스트림 (Conversation Stream)
- 좌측 패널에 채팅 형태로 실시간 표시
- 고객(파란색) / 상담사(초록색) 구분
- 중간 인식 결과는 회색으로 실시간 업데이트
- 자동 스크롤 + "최신으로 이동" 버튼
- 고객 정보 카드 (이름, 마스킹된 전화번호, 경과 시간)

### 3. AI 인텔리전스 패널 (중앙 패널)
- 대화 중 AI FAB 버튼(또는 Ctrl+Shift+A)으로 분석 트리거
- 대화 전문에서 키워드 자동 추출 (요금제, 로밍, 분실, 멤버십 등 12개 카테고리)
- 키워드 우선순위 알고리즘으로 가장 관련성 높은 주제 선택
- 분석 결과 표시:
  - 검색 쿼리 (AI가 생성한 검색어)
  - 상담 요약 (맥락 기반)
  - 추천 응대 스크립트 (고객 감정/요청에 따라 동적 생성)
  - 관련 문서 목록 (관련도 점수 포함, 클릭 시 문서 뷰어에서 열림)
- 분석 히스토리 탭 (한 통화 내 여러 번 분석 기록)

### 4. 문서 검색 및 뷰어 (우측 패널)
- 44개 마크다운 지식베이스 파일 전문 검색
- 카테고리 필터링 (요금, 로밍, 디바이스, 매장 등)
- 검색 결과에 매칭 건수, 관련 스니펫 표시
- 문서 뷰어:
  - 마크다운 풀 렌더링 (테이블, 코드블록, 체크리스트 등)
  - SK텔레콤 브랜딩 (#E4002B 레드)
  - 문서 간 내부 링크 네비게이션
- 지식베이스 내용: 5G/LTE 요금제, 로밍, T멤버십, 디바이스 스펙, 매장 정보, 보험/보증, 결합상품 등

### 5. 통화 관리
- 통화 상태 관리: 대기 → 연결중 → 통화중 → 보류 → 후처리
- 시작/종료/보류/재개 컨트롤
- 세션 ID 자동 생성, 고객 정보 자동 매칭
- 통화 시간 실시간 카운터

### 6. 통화 종료 후처리 (Wrap-Up)
- AI가 대화 전문 기반으로 상담 요약 자동 생성 (마크다운)
- 고객에게 발송할 SMS 템플릿 자동 생성
- PII 마스킹 자동 적용:
  - 전화번호, 신용카드, 주민번호, 이메일, 생년월일, 계좌번호 감지 및 마스킹
  - 보안 상태 배지 표시
- 요약 및 SMS 편집 가능

### 7. 상담 이력
- 과거 상담 검색 (고객명, 내용)
- 고객명, 마스킹된 전화번호, 요약, 통화 시간 표시
- 개별 상담 상세 보기 (전체 대화 내용)

### 8. 고객 서비스 관리 (/advanced)
- 6개 탭 기반 관리 인터페이스 (프로필, 요금제, 로밍, 부가서비스, 할인, 요금 시뮬레이터)
- 데모 시나리오 드롭다운으로 다양한 고객 상황 전환 가능
- 각 테이블에서 현재 가입 항목 표시, 변경/해지 분석 트리거

### 9. 해지 영향 분석 (Cancellation Analysis)
- 요금제·부가서비스·할인 각 항목에 "해지 분석" 버튼
- 해지 시 **월별 요금 변동 차트** (12개월 예측)
- 누적 비용 영향 시각화 (해지 전 vs 후 비교 차트)
- 연관 서비스 영향 경고 (예: 가족결합 할인 해지 시 다른 가족 회선도 영향)
- 위약금·잔여 약정 자동 계산
- 대안 요금제 추천 제시

### 10. 요금 시뮬레이터 (Plan Calculator)
- 요금제 검색 + 카드형 선택 UI
- **고객 사용 이력 시각화**:
  - 최근 6개월 월별 데이터 사용량 바 차트
  - 월평균/최대 데이터·통화량 지표
  - 요금제 변경 히스토리 (가입 이후 전체)
- **AI 추천**: 대화 키워드 + 실 사용 패턴(평균·최대 데이터) 기반 최적 요금제 추천 (추천 사유 표시)
- 부가서비스·할인 체크박스 (카테고리별 그룹핑, 약정/가족결합 배타 선택)
- **실시간 요금 계산**: 기본료 + 부가서비스 - 할인 = 예상 월 요금
- 현재 요금제 대비 절약/추가 금액 비교

### 11. 데모 시나리오 시스템
- 복수 시나리오 프리셋 (고객별로 다른 요금제/부가서비스/할인 구성)
- 시나리오 전환 시 요금제 테이블, 부가서비스, 할인 데이터가 연동 변경
- 해지 분석용 확장 데이터 포함 (위약금, 잔여 약정, 연관 서비스 등)

### 12. 상담 시나리오 (지식베이스 내)
- 시나리오 1: 요금제 변경 상담 (5G Prime → 5G Slim, 가족결합 포함)
- 시나리오 2: 분실폰 보험 접수

## 상담사 워크플로우 (10단계)

```
① 로그인 → ② 상담 시작 → ③ 실시간 대화 (STT)
→ ④ AI 분석 (맥락·문서·스크립트)
→ ⑤ 문서 참조
→ ⑥ 고객 서비스 관리 (요금제/부가서비스/할인 + 해지분석 + 요금 시뮬레이터)
→ ⑦ 상담 종료 → ⑧ AI 요약 + SMS 발송
→ ⑨ 마무리 완료 → ⑩ 상담 이력 조회
```

## 요청사항

위 서비스는 SK텔레콤 USIM 해킹 사태 이후 **고객 신뢰 회복**을 위한 AICC 데모입니다.

현재 기능은 상담사 업무 효율화에 초점이 맞춰져 있습니다. 여기에 **고객 신뢰 회복에 직접적으로 기여하는 새로운 기능**을 제안해주세요.

조건:
1. 기존에 이미 있는 기능(요약, PII마스킹, 문서검색, STT, 해지분석, 요금시뮬레이터, AI추천 등)은 제외하고 완전히 새로운 기능만 제안
2. 단순한 개선(UI 개선, 검색 고도화 등)이 아니라, **데모에서 관객이 "와" 할 만한 창의적이고 획기적인 아이디어**
3. **고객의 시각에서** 신뢰가 회복되는 경험이어야 함 (상담사 편의가 아닌 고객 체감)
4. 현재 기술 스택(Next.js, OpenAI API, Supabase)으로 구현 가능한 범위
5. 각 아이디어마다 다음을 포함:
   - 기능 이름
   - 한 줄 컨셉 ("핵심 메시지")
   - 구체적인 작동 방식
   - 데모에서 어떻게 보여줄 것인지
   - 왜 이것이 고객 신뢰 회복에 효과적인지

5~7개 아이디어를 제안해주세요.
```

## 프롬프트 끝
