# 실시간 자막 테스트 가이드

## 🎯 유튜브처럼 실시간 자막 생성

말하는 **즉시** 화면에 자막이 표시됩니다.

## 테스트 방법

### 1. Chrome 브라우저로 접속

**중요**: 이 기능은 **Chrome** 또는 **Edge** 브라우저에서만 작동합니다.
- Safari: 제한적 지원
- Firefox: 지원 안 됨

### 2. 대시보드 접속

```
http://localhost:3000/dashboard
```

### 3. 실시간 모드 선택

1. 상단 탭에서 **"실시간"** 클릭
2. **"상담 시작"** 버튼 클릭
3. **"고객"** 또는 **"상담사"** 마이크 버튼 클릭

### 4. 마이크 권한 허용

브라우저 상단에 마이크 권한 요청이 나타나면 **"허용"** 클릭

### 5. 말하기 시작

**즉시** 화면에 자막이 나타납니다!

## 📊 실시간 자막 작동 방식

### 말하는 동안 (Interim Results)

```
사용자: "안녕하"
화면: [회색, 점선 테두리] 안녕하 ● 실시간 인식 중...

사용자: "안녕하세요"
화면: [회색, 점선 테두리] 안녕하세요 ● 실시간 인식 중...

사용자: "안녕하세요 고객님"
화면: [회색, 점선 테두리] 안녕하세요 고객님 ● 실시간 인식 중...
```

### 말이 끝나면 (Final Result)

```
화면: [검정, 실선] 안녕하세요 고객님 12:34:56
```

## 🔍 디버깅

### 브라우저 콘솔 확인

1. **F12** 또는 **Cmd+Option+I** (Mac)로 개발자 도구 열기
2. **Console** 탭 선택
3. 말하면 다음과 같은 로그가 실시간으로 나타납니다:

```javascript
Speech recognition started
Speech started
Speech recognition result: [...]
[customer] Interim: 안녕하
[customer] Interim: 안녕하세요
[customer] Interim: 안녕하세요 고객님
Speech ended
[customer] Final: 안녕하세요 고객님
```

### 문제 해결

#### 1. "이 브라우저는 음성 인식을 지원하지 않습니다"
→ Chrome 브라우저를 사용하세요

#### 2. 마이크 아이콘이 비활성화됨
→ 브라우저 설정에서 마이크 권한을 확인하세요
→ `chrome://settings/content/microphone`

#### 3. 자막이 나타나지 않음
→ F12 콘솔에서 에러 메시지 확인
→ 마이크가 제대로 연결되었는지 확인
→ 인터넷 연결 확인 (Google 서버 사용)

#### 4. 말이 끝나야만 자막이 나타남
→ Chrome이 아닌 다른 브라우저를 사용 중일 수 있습니다
→ `recognition.interimResults = true` 설정 확인

## 🎨 UI 표시 방식

### Interim (실시간 인식 중)
- 회색 텍스트
- 점선 테두리 (파란색/초록색)
- 투명도 70%
- 깜빡이는 커서 애니메이션
- "● 실시간 인식 중..." 표시

### Final (확정)
- 검정 텍스트
- 실선 테두리
- 투명도 100%
- 타임스탬프만 표시

## 🚀 성능

- **지연 시간**: < 200ms (즉시)
- **업데이트 빈도**: 실시간 (말하는 동안 계속 업데이트)
- **정확도**: Google Speech API 수준 (매우 높음)
- **비용**: 무료

## 📝 예상 결과

말하기 시작하면:

1. **0.1초 후**: 첫 단어가 화면에 나타남 (회색)
2. **계속 말하면**: 문장이 실시간으로 늘어남 (회색)
3. **말을 멈추면**: 2초 후 확정 (검정)
4. **다시 말하면**: 새로운 문장이 즉시 시작 (회색)

## ⚡ 핵심 차이점

